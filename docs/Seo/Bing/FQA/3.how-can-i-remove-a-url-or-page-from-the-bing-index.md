# 如何将网站上的页面从 Bing 索引中移除？

本文介绍站长或网站所有者可用的不同方法，将已存在站点上的 URL 从 Bing 索引中移除。

> **注意：** 如果您并非要移除页面的站点的站长或所有者，请联系该站点的站长，以便他们下架内容。如果内容已经从原站点删除，您可以使用 [内容移除工具](https://www.bing.com/webmasters/help/?topicid=cb6c294d) 向 Bing 请求将页面从搜索结果中删除。

## 方法一：从站点删除页面并返回 404

自然移除 URL 的方法是将该页面从站点中删除，使该 URL 返回 [404（未找到）](http://en.wikipedia.org/wiki/HTTP_404) 或 410（已移除）HTTP 状态码。我们在重新抓取页面并确认其已从站点永久移除后，会将该页面从索引中删除。

**注意：** 为了让 Bing 检测到页面已被删除并返回 404 或 410，Bingbot 必须能够访问该 URL，因此不要通过 `robots.txt` 阻止该 URL 被重新抓取。

## 方法二：在页面上添加 NOINDEX 元标签

如果您无法或不想按方法一删除页面，可以在页面 HTML 源码中添加 robots 元标签。Bing 会遵循 meta robots 标签，向页面源添加 `<meta name="robots" content="noindex">`，在页面被重新抓取并处理后将从索引中移除。示例页面头部如下：

```html
<html><head>
<meta name="robots" content="noindex" />
(…其余内容…)
</head>
<body>…</body>
</html>
```

重新抓取和处理可能需要一些时间。同时，请确保不要通过 `robots.txt` 或其他方式禁止抓取该 URL；我们需要能够获取页面以识别 NOINDEX 标签。

**注意：** 如果您要移除的 URL 不是网页而是例如 PDF，您可以在响应头中包含 `X-Robots-Tag: noindex`：

```http
HTTP/1.1 200 OK
Cache-Control: private
Content-Type: text/html; charset=utf-8
X-Robots-Tag: noindex
```

## 方法三：使用“屏蔽 URL”工具移除 URL

如果您急于将某个 URL 从搜索结果中移除，但又不想等待自然重抓取过程，您可以在 Bing Webmaster Tools 中使用 [屏蔽 URL](https://www.bing.com/webmasters/help/?topicid=264e560b) 工具。该操作可在 24 小时内阻止 URL 在搜索结果中显示。请注意，屏蔽 URL 仅为临时措施，屏蔽记录会在每 90 天后失效。这样可以为您争取时间：在此期间，您可以从站点上移除该页面或向页面添加 NOINDEX 元标签，以彻底从 Bing 索引中删除该 URL。
