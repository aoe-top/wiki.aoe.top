## Bing 视频 Feed 类型和要求</table>



我们建议您使用 Bing 媒体格式来整合您的视频内容，遵循这些 [指南](http://www.rssboard.org/media-rss)。如果您有现有的视频专用站点地图或 Yahoo mRSS，您也可以使用它们 - 这取决于您的便利性。您可以在我们的博客文章 [这里](http://blogs.bing.com/webmaster/2012/04/24/video-killed-the-radio-star-but-we-still-want-your-videos/) 找到我们支持的格式。<p>如果您的网站包含视频内容，我们建议您向 Bing 提供视频 Feed，这可以作为您拥有的任何

  通用 XML 站点地图或 RSS Feed 的补充。视频专用 Feed 可以作为常规视频

我们目前支持三种类型的视频 Feed：完整回目录 Feed（完整 Feed）、增量更新 Feed（增量 Feed）和过期 Feed。它们定义如下：  发现机制的有效补充，因为它确保 Bing 知道您的所有视频内容，即不仅仅是托管视频的 URL，

  还包括视频的丰富元数据，如标题、缩略图位置、视频位置等。

- **完整 Feed** 用于确保 Bing 能够获取和索引所有合作伙伴视频及其正确的元数据。此 Feed 通常在初始索引时处理一次，但应该保持更新，包含您希望被 Bing 收录的所有视频。

</p>

- **增量 Feed** 用于提供更新的视频内容。在这里，您应该指定更新频率，例如每小时、每天或每周。您必须添加 `<changefreq>daily</changefreq>` 来确定网站信息初始阶段的更新频率。将每个增量 Feed 的大小限制为最大 10MB。我们建议您将所有增量文件归档在一个摘要页面中，并在大小达到其限制时添加新的文件。<p>您只需要按照下面详细说明的规范为您的网站提供合适的 Feed。我们还在页面底部包含了一些

  示例来帮助您了解所需的格式。</p>

- **过期 Feed** 用于识别已发布但已过期且需要从 Bing 索引中删除的视频。仅对最新的过期视频使用过期 Feed。过期 Feed 中只需要 `<link>` 字段。

<p>Once you create the feed/feeds, you can submit them using the <a href="https://www.bing.com/webmasters/help/sitemaps-3b5cf6ed">Sitemap</a> feature.You can also email the feed location/locations to <a href="mailto:bingfeed@microsoft.com">Bing Content Feed Support</a>.</p>

> **注意**<p>

> 

> 删除视频的理想方法是确保在原始 Feed 中包含 `<expireDate>` 标签，由 Bing 爬取并索引。此值是您希望 Bing 从搜索结果中移除视频的日期。我们理解并非所有内容过期在发布时都是已知的，因此过期 Feed 允许您移除没有特定"过期日期"的视频，以避免 Bing 搜索结果向用户提供过期视频。</p>
